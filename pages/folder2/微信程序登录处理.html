# 微信小程序登录理解

根据官方文档说明，wx.login所获得的code值仅在5分钟内有效，根据获得的code值结合小程序AppId和小程序秘钥AppSecret来解析code获得用户在这个小程序下的唯一标识openId,和本次会话秘钥session_key，后端将这两个处理为自定义登录状态token,由于（实际上）每一次login获得的openid不变，session_key一直在变化，从而token也一直在变化。但是变化的触发只有在每一次登录时获取code。

官方文档也提到，调用wx.login会引起登录态刷新。

用户每次登录小程序（包括第一次使用）及点击小程序的每个页面的时候，我们如何判断他当前的登录状态是否过期?如果过期，如何重新获取用户信息并发送至开发者服务器更新用户信息，以及设置新的用户登录状态？

为什么要使用token值来作为前后端交互的桥梁，而不是直接使用openid来进行交互？

首先，我们应该清楚地知道token值和openid都能够在该微信小程序下唯一标识用户。我们根据wx.login获得的code值，在传回服务器（后端）解析后，获得当前登录用户的唯一身份标识openId和本次会话秘钥session_key，直接返回openid，存储在前端缓存中来标识用户，存在一定的风险，这也是在微信开发文档中官方不建议的。我们可以通过openid会话密匙来组建我们自己的登录状态token，并在前端缓存，即便token泄露，它也解不开openid不知道用户是哪一个，安全性更高。

出于安全性和效率性的问题，也是目前普遍存在的一种网页端设计思路->登录超时，即我们在登录后，没有操作在一个页面停留时间过长，后台服务器就应该具备将我们登录状态强制下线的操作。在新的网络请求发起之前，应该验证用户是否登录，以及登录状态是否正常。不存在登录已经在前端缓存中记录，即缓存中有token值，但是登录状态的不正常（生存周期已经结束）但是依旧可以发起网络请求，这就有些不合理不是？你都被服务器那边给下线了怎么还可以请求，如果服务器端没有相应的处理操作的话，意味着获去一个人的token值就可以通过他来获取这个人的信息完成一系列操作，因为数据库中的token值只有在每一次重新登录后才会刷新。

针对这种情况，微信小程序中有API专门对我们的当前已经登录的用户校验其登录状态是否正常：

wx.checkSession({
 success: function () {
 //session_key 未过期，并且在本生命周期一直有效
 return ;
 },
 fail: function () {
 // session_key 已经失效，需要重新执行登录流程
 wx.navigateTo({
  url: "/pages/authorize/index"
 })
 }
})

# List类型数据判断为不为空

我们在调用一个方法返回数据列表时，通常会将返回列表逐条注入实体对象中，并建立以该对象为类型的集合List。

           List<Order>orders= orderDao.queryOrderByStation(token,station);
           if(orders!=null){
               System.out.println("数据不为空");
               return new ResponseData(ResponseCode.SUCCESS,orders);
           }else {
               System.out.println("数据为空");
               return  new ResponseData(ResponseCode.ERROR,"未获取到数据");
           }

           List<Order>orders= orderDao.queryOrderByStation(token,station);
           if(orders!=null && !orders.isEmpty()){
               System.out.println("数据不为空");
               return new ResponseData(ResponseCode.SUCCESS,orders);
           }else {
               System.out.println("数据为空");
               return  new ResponseData(ResponseCode.ERROR,"未获取到数据");
           }

根据这两个代码块我想大家就会明白了，我们判断一个集合为不为空，应该从两方面考虑。

一、集合是否有实例化分配空间，不是null。

二、集合分配了空间，但其内元素个数是否不为0。

if(list == null || list.size() ==0 ){
　　//这里写list为空的情况
}else{
　　//这里写list不为空的情况
   //比如可以做取值的一些操作
}

if(list != null && !list.isEmpty()){
　　//这里写list不为空的情况
}else{
　　//这里写list为空的情况
}

# 页面渲染问题

基本思路和设置：页面设置变量station作为登录状态的标识。0：未登录；1：登录。前端html中按钮是否可用依赖于对登录状态的判断。

 <view>
  <button class="login" type="warn" plain="true" size="mini" bindtap="toLogin" disabled="{{station=='1'?true:false}}">登录</button>
  <button class="quit" type="warn" plain="true" size="mini" bindtap="toQuit" disabled="{{station=='0'?true:false}}">退出</button>
</view>
  <button class="change" plain="true" type="warn" size="mini" bindtap="toChange" disabled="{{station=='0'?true:false}}">修改密码</button>

# onLoad生命周期函数
# 触发时机
  • 第一次初始化小程序页面时触发（小程序页面初始化，运行的最早的生命周期函数）
# 触发条件

  • 小程序当前页面第一次被加载时(并且小程序页面栈中没有当前页面的缓存)

  • # 注意点

    • 使用wx.navigateBack跳转到上个页面，不会触发上个页面onLoad

      wx.navigateBack({
    delta: 0
})
//因为上一个页面还在，页面栈中（可以理解为缓存中）

    • 跳转到tabbar页面也不会触发tabbar页面的onLoad

       wx.switchTab({
     url: '/pages/index/index',
 })

    • 无法通过getCurrentPage()获取当前页面，因为page对象还未生成

    • 唯一可以获取传递参数的生命周期

    # 使用场景
    • 请求数据
      • 不用每次回到这个页面都要刷新的数据
    • 例如：轮播图、首页商品、评论
    • 第一次进入页面就要做的事情
      • 判断登录
      • 权限校验
      • 数据处理
    # onShow生命周期函数
    # 触发时机
    • 第一次页面显示（小程序页面显示初始化，运行的最早的生命周期函数）
    • 后台切到前台 （小程序切到手机后台，短时间内再打开，长时间小程序会被手机杀死）
    • 页面跳转（所以跳转都可以）
    # 触发条件
    • 小程序当前页面第一次被加载时
    • 小程序当前页面再一次显示（程序页面栈中有当前页面的缓存）
    # 注意点
    • 使用wx.previewImage放大查看图片，会触发页面的onShow
    # 使用场景
    • 请求数据
      • 来回跳转页面需要实时更新的数据
      • 后台切换到前台，需要做的事情
