# Controller层的注解

# @ResponseBody

注解的作用是将controller的方法返回对象通过适当的转换器转换为指定的格式之后写入response对象的body区，通常返回JSON数据或者是XML数据。

# @RequestBody

@RequestBod将HTTP请求正文插入方法中，使用合适的HttpMessageConverter将请求体写入某个对象，类似于@Autowired的逆向使用。

后台 Controller类中对应的方法：
@RequestMapping("/login.do")
@ResponseBody
public Object login(String name, String password, HttpSession session) {
	user = userService.checkLogin(name, password);
	session.setAttribute("user", user);
	return new JsonResult(user);
}
 
@RequestBody是作用在形参列表上，用于将前台发送过来固定格式的数据【xml格式 或者 json等】封装为对应的 JavaBean 对象，
封装时使用到的一个对象是系统默认配置的 HttpMessageConverter进行解析，然后封装到形参上。
如上面的登录后台代码可以改为：
@RequestMapping("/login.do")
@ResponseBody
public Object login(@RequestBody User loginUuser, HttpSession session) {
	user = userService.checkLogin(loginUser);
	session.setAttribute("user", user);
	return new JsonResult(user);
}

# @configurationProperties

使用@ConfigurationProperties 注解，我们可以方便的获取这些配置在配置文件(.properties和.yml)中的参数属性，需要注意的是类的属性名必须与配置文件中prefix（" "）对应前缀的外部名相同，且类必须由所有参数的setter方法，这里建议使用lombok中的@Data注解。

# 绝对路径和相对路径的区别

绝对路径是不加参考的全局路径，能够唯一找到该资源路径，是从根路径出发的路径。

相对路径一般是相对于外层文件夹或者资源包的路径表示方法，如果被参考问价发生位置移动，则参考文件路径也会变更。
